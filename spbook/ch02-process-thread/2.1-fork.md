# 2.1 程序與 fork()

## 什麼是程序？

程序（Process）是一個正在執行中的程式實體。每個程序包含：

- **Text Segment**：程式碼（唯讀）
- **Data Segment**：已初始化的全域變數
- **BSS Segment**：未初始化的全域變數
- **Heap**：動態配置的記憶體（`malloc` / `new`）
- **Stack**：區域變數、函式呼叫框架
- **CPU 狀態**：暫存器、Program Counter
- **核心資源**：檔案描述符、PID、訊號處理器

## fork() — 建立子程序

`fork()` 是 Linux 上建立新程序的唯一方式（除了 `clone()` 等變體）。它會複製當前呼叫的程序：

```
   fork() 之前
   PID=5012 (only one process)
       │
       ▼ fork()
   ┌──┴──┐
   │     │
   ▼     ▼
PID=5012  PID=5013
(父程序)  (子程序)
```

### fork 的回傳值規則

| 回傳值 | 語意 |
|--------|------|
| `-1` | fork 失敗 |
| `0` | 目前在子程序中 |
| `>0` | 目前在父程序中，值為子程序 PID |

### 程式範例

```c
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    printf("fork() 之前，PID=%d\n", getpid());

    pid_t pid = fork();

    if (pid < 0) {
        perror("fork 失敗");
        return 1;
    } else if (pid == 0) {
        // ── 子程序 ──
        printf("[子程序] PID=%d, 父PID=%d\n",
               getpid(), getppid());
        printf("[子程序] 工作中...\n");
        sleep(1);
        printf("[子程序] 完成！\n");
        return 0;
    } else {
        // ── 父程序 ──
        printf("[父程序] PID=%d, 子程序PID=%d\n",
               getpid(), pid);
        printf("[父程序] 等待子程序結束...\n");
        wait(NULL);
        printf("[父程序] 子程序已結束，繼續執行\n");
    }
    return 0;
}
```

**執行結果：**

```
$ gcc -o fork_demo fork_demo.c && ./fork_demo
fork() 之前，PID=5012
[父程序] PID=5012, 子程序PID=5013
[父程序] 等待子程序結束...
[子程序] PID=5013, 父PID=5012
[子程序] 工作中...
[子程序] 完成！
[父程序] 子程序已結束，繼續執行
```

> 💡 **寫時複製（Copy-on-Write）**：fork() 並不會立即複製所有記憶體，而是讓父子共享相同的物理頁面。只有當其中一方**寫入**時，才真正複製該頁面。這大幅提升了 fork() 的效能。

## 父程序監控：wait()

父程序應使用 `wait()` 或 `waitpid()` 等待子程序結束，避免產生**殭屍程序（Zombie Process）**：

```c
int status;
pid_t done = wait(&status);           // 等待任意子程序
pid_t done = waitpid(pid, &status, 0); // 等待特定子程序
```

## exec() 家族

`fork()` + `exec()` 是 Unix 啟動新程式的標準模式：

```c
pid_t pid = fork();
if (pid == 0) {
    // 子程序：載入新程式取代自身
    execlp("ls", "ls", "-la", NULL);
    perror("exec 失敗");
    exit(1);
}
wait(NULL);
```

## 思考題

> **Q**：fork() 後，父子程序各自修改同一個全域變數，會互相影響嗎？  
> **A**：不會。fork() 後，父子程序有獨立的虛擬記憶體空間（Copy-on-Write）。一方修改變數時，OS 會複製該記憶體頁，雙方各自擁有獨立的副本。
