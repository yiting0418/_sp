# 2.2 執行緒（Thread）

## 什麼是執行緒？

執行緒（Thread）是同一程序內的並行執行單元。同一程序的執行緒**共享記憶體空間**（Heap、全域變數、檔案描述符），但各自有獨立的：

- Stack（區域變數）
- 暫存器狀態（包含 Program Counter）
- Signal Mask

## 程序 vs 執行緒

| 比較項目 | 程序 (Process) | 執行緒 (Thread) |
|---------|---------------|----------------|
| 記憶體空間 | 各自獨立 | 共享同一程序空間 |
| 建立成本 | 高（需複製資源） | 低（共享大部分資源） |
| 通訊方式 | IPC（管道、Socket） | 直接讀寫共享記憶體 |
| 安全性 | 高（互不干擾） | 低（需同步機制） |
| 崩潰影響 | 只影響自己 | 可能讓整個程序崩潰 |

## pthread 範例

```c
#include <pthread.h>
#include <stdio.h>
#include <unistd.h>

void *worker(void *arg) {
    int id = *(int *)arg;
    printf("[執行緒 %d] 開始執行\n", id);
    for (int i = 1; i <= 3; i++) {
        printf("[執行緒 %d] 步驟 %d/3\n", id, i);
        usleep(100000);  // 0.1 秒
    }
    printf("[執行緒 %d] 完成！\n", id);
    return NULL;
}

int main() {
    pthread_t t1, t2, t3;
    int ids[] = {1, 2, 3};

    printf("主執行緒：建立 3 個工作執行緒\n");
    pthread_create(&t1, NULL, worker, &ids[0]);
    pthread_create(&t2, NULL, worker, &ids[1]);
    pthread_create(&t3, NULL, worker, &ids[2]);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    pthread_join(t3, NULL);

    printf("主執行緒：所有工作完成！\n");
    return 0;
}
// 編譯：gcc -pthread -o thread_demo thread_demo.c
```

**執行結果：**

```
$ gcc -pthread -o thread_demo thread_demo.c && ./thread_demo
主執行緒：建立 3 個工作執行緒
[執行緒 1] 開始執行
[執行緒 2] 開始執行
[執行緒 3] 開始執行
[執行緒 1] 步驟 1/3
[執行緒 2] 步驟 1/3
[執行緒 3] 步驟 1/3
... (交錯執行)
[執行緒 1] 完成！
[執行緒 2] 完成！
[執行緒 3] 完成！
主執行緒：所有工作完成！
```

可以看到三個執行緒**交錯執行**，順序不固定——這是 OS 排程器的決策。

## 執行緒的生命週期

1. **建立**：`pthread_create()` → 建立新執行緒，立即開始執行
2. **執行**：執行緒執行指定的 `start_routine`
3. **結束**：執行緒回傳或呼叫 `pthread_exit()`
4. **等待**：其他執行緒呼叫 `pthread_join()` 回收資源

## 思考題

> **Q**：為什麼執行緒的建立成本比程序低？  
> **A**：執行緒共享父程序的記憶體空間（Heap、全域變數、檔案描述符），不需要像 fork() 那樣複製完整的位址空間。只需要建立獨立的 Stack 和暫存器狀態即可，因此更輕量快速。
