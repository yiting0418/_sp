# 5.1 迷你 Shell 實作

## Shell 的 REPL 主迴圈

Shell（命令列解譯器）的核心是 **REPL（Read-Eval-Print-Loop）** 迴圈：

```
  loop:
    1. Read   ── 顯示提示符，讀取使用者輸入
    2. Eval   ── 詞法分析，切割命令與參數
    3. Execute ── fork() + execvp() 執行命令
    4. Wait   ── waitpid() 等待子程序完成
    5. ← 回到 1
```

## 完整實作

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/wait.h>

#define MAX_ARGS 64
#define MAX_LINE 256

// 詞法分析：切割輸入為 argv
int tokenize(char *line, char *argv[]) {
    int argc = 0;
    char *tok = strtok(line, " \t\n");
    while (tok && argc < MAX_ARGS - 1) {
        argv[argc++] = tok;
        tok = strtok(NULL, " \t\n");
    }
    argv[argc] = NULL;
    return argc;
}

void execute(char *argv[]) {
    // 內建命令：cd
    if (strcmp(argv[0], "cd") == 0) {
        const char *dir = argv[1] ? argv[1] : getenv("HOME");
        if (chdir(dir) != 0) perror("cd");
        return;
    }
    // 內建命令：exit
    if (strcmp(argv[0], "exit") == 0) {
        printf("Bye!\n"); exit(0);
    }

    // 外部命令：fork + exec
    pid_t pid = fork();
    if (pid == 0) {
        execvp(argv[0], argv);
        fprintf(stderr, "minish: 找不到命令: %s\n", argv[0]);
        exit(1);
    } else if (pid > 0) {
        int status;
        waitpid(pid, &status, 0);
    }
}

int main() {
    char line[MAX_LINE];
    char *argv[MAX_ARGS];
    printf("=== 迷你 Shell（輸入 exit 離開）===\n");
    while (1) {
        char cwd[256];
        getcwd(cwd, sizeof(cwd));
        printf("\033[32m%s\033[0m $ ", cwd);
        fflush(stdout);
        if (!fgets(line, sizeof(line), stdin)) break;
        if (line[0] == '\n') continue;
        int argc = tokenize(line, argv);
        if (argc == 0) continue;
        execute(argv);
    }
    return 0;
}
```

**執行結果：**

```

```

## 核心概念

| 概念 | 在本實作中的展現 |
|------|----------------|
| **fork()** | 建立子程序執行外部命令 |
| **execvp()** | 在子程式中載入並執行新程式 |
| **waitpid()** | 父程序等待子程序完成 |
| **chdir()** | 實作內建 `cd` 命令 |
| **strtok()** | 詞法分析：切割命令與參數 |
| **getcwd()** | 取得當前工作目錄 |

## 進階擴充方向

1. **管線（Pipe）**：支援 `cmd1 | cmd2`
2. **重定向**：支援 `>`、`<`、`>>`
3. **背景執行**：支援 `&`
4. **歷史指令**：上下鍵瀏覽歷史
5. **自動補全**：Tab 補全檔案名稱
6. **Wildcard 展開**：支援 `*`、`?`

## 思考題

> **Q**：為什麼外部命令需要用 `fork()` + `exec()` 而不是直接在 Shell 程序中執行？  
> **A**：`exec()` 會將當前程序的程式碼完全替換為新程式。如果不在子程序中執行，Shell 自身的程式碼會被覆蓋，執行完命令後就無法回到 Shell 主迴圈了。因此必須先 `fork()` 複製自身，再在子程序中 `exec()` 執行新程式，父程序則等待子程序結束後繼續。
