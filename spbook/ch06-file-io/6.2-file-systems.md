# 6.2 檔案系統與 inode

## inode

inode（index node）是 Unix 檔案系統的核心資料結構。每個檔案和目錄在磁碟上都對應一個 inode，儲存檔案的中繼資料：

```
inode 結構（簡化）
┌──────────────────────────┐
│ 檔案類型與權限 (mode)     │
│ 擁有者 (UID/GID)          │
│ 檔案大小                  │
│ 時間戳 (atime/mtime/ctime)│
│ 參考計數 (link count)     │
│ 直接區塊指標 (12 個)      │
│ 間接區塊指標              │
│ 雙重間接區塊指標          │
│ 三重間接區塊指標          │
└──────────────────────────┘
```

### 檔案系統架構

```
┌────────────────────────────────────────┐
│ Boot Block | Superblock | inode list | Data blocks │
└────────────────────────────────────────┘

Superblock: 檔案系統的中繼資料（大小、inode 數量、空閒區塊等）
inode list: 所有 inode 的陣列
Data blocks: 實際檔案內容儲存在這裡
```

## 硬連結與符號連結

### 硬連結（Hard Link）

一個 inode 可以有多個目錄項目（Directory Entry）指向它：

```
inode #42 (link count = 3)
    ↑          ↑          ↑
"file.txt"  "backup"  "docs/data.txt"
```

- 所有硬連結都指向**同一個 inode**
- 刪除其中一個不會刪除檔案（僅 link count -1）
- link count 為 0 時，檔案才會被真正刪除
- 不能跨檔案系統，不能連結目錄

### 符號連結（Symbolic Link，又稱 Soft Link）

符號連結是一個**特殊的檔案**，其內容是目標路徑：

```
"link.txt" → "file.txt"
   ↑
symbolic link（自己的 inode，內容儲存 "file.txt"）

如果 "file.txt" 被刪除，"link.txt" 變成懸空連結（dangling link）
```

### 差異比較

| 特性 | 硬連結 | 符號連結 |
|------|--------|---------|
| inode | 與目標相同 | 自己的 inode |
| 跨檔案系統 | ❌ 不可 | ✅ 可 |
| 連結目錄 | ❌ 不可 | ✅ 可 |
| 懸空（目標刪除） | ✅ 檔案仍在 | ❌ 變成 dead link |
| 建立命令 | `ln src dst` | `ln -s src dst` |

## 目錄操作

```c
#include <stdio.h>
#include <dirent.h>
#include <sys/stat.h>

void list_dir(const char *path) {
    DIR *dir = opendir(path);
    if (!dir) { perror("opendir"); return; }

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        // 跳過 . 和 ..
        if (strcmp(entry->d_name, ".") == 0 ||
            strcmp(entry->d_name, "..") == 0) continue;

        struct stat st;
        char full[1024];
        snprintf(full, sizeof(full), "%s/%s", path, entry->d_name);
        stat(full, &st);

        printf("%c %s (%ld bytes)\n",
               S_ISDIR(st.st_mode) ? 'D' : 'F',
               entry->d_name, st.st_size);
    }
    closedir(dir);
}
```

## VFS（Virtual File System）

Linux 的 VFS 層讓不同檔案系統可以統一管理：

```
系統呼叫：open() read() write() close()
            │
          VFS 層（通用檔案操作）
            │
  ┌─────────┼────────────┐
  │         │             │
 ext4     xfs          tmpfs       ← 具體檔案系統驅動
  │         │             │
  └─────────┴─────────────┘
            │
        磁碟 / RAM
```

## 思考題

> **Q**：`mv` 命令在同一個檔案系統內移動時，為什麼速度非常快？  
> **A**：`mv` 在同一個檔案系統內操作時，只是修改目錄項目（Directory Entry）中的檔名或路徑，指向**同一個 inode**。實際檔案內容完全不需要搬動。只有跨檔案系統時，才需要複製內容。
