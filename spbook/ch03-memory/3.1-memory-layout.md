# 3.1 程序記憶體佈局

## 虛擬記憶體空間

每個 Linux 程序的虛擬記憶體空間由以下區段組成（從低位址到高位址）：

```
  高位址 0xFFFFFFFF
  ┌──────────────────────────────────────┐
  │ Stack（堆疊）          ↓ 往下成長      │  ← 區域變數、函式呼叫框架
  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
  │   （未使用空間）                     │
  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
  │ Heap（堆積）           ↑ 往上成長      │  ← malloc / new
  ├──────────────────────────────────────┤
  │ BSS Segment                          │  ← 未初始化的全域變數
  ├──────────────────────────────────────┤
  │ Data Segment                         │  ← 已初始化的全域/靜態變數
  ├──────────────────────────────────────┤
  │ Text Segment（唯讀）                 │  ← 程式碼
  └──────────────────────────────────────┘
  低位址 0x00000000
```

### 各區段功能

| 區段 | 存放內容 | 成長方向 | 存取權限 |
|------|---------|---------|---------|
| **Text** | 程式碼、唯讀常數 | 固定 | R+X |
| **Data** | 已初始化的全域/靜態變數 | 固定 | R+W |
| **BSS** | 未初始化的全域/靜態變數 | 固定 | R+W |
| **Heap** | 動態配置的記憶體 | ↑ 往上 | R+W |
| **Stack** | 區域變數、參數、回傳位址 | ↓ 往下 | R+W |

### 觀察程式：印出各區段位址

```c
#include <stdio.h>
#include <stdlib.h>

int global_init = 42;      // Data Segment
int global_uninit;         // BSS Segment

void show_addresses() {
    int stack_var = 10;    // Stack
    static int static_var = 99;  // Data Segment

    int *heap_var = (int *)malloc(sizeof(int));
    *heap_var = 777;       // Heap

    printf("=== 記憶體位址分佈 ===\n");
    printf("[Text]  函式位址:     %p\n", (void*)show_addresses);
    printf("[Data]  全域已初始化: %p  值=%d\n",
           (void*)&global_init, global_init);
    printf("[Data]  靜態變數:     %p  值=%d\n",
           (void*)&static_var, static_var);
    printf("[BSS]   全域未初始化: %p  值=%d\n",
           (void*)&global_uninit, global_uninit);
    printf("[Heap]  malloc 配置:  %p  值=%d\n",
           (void*)heap_var, *heap_var);
    printf("[Stack] 區域變數:     %p  值=%d\n",
           (void*)&stack_var, stack_var);

    free(heap_var);
}

int main() {
    show_addresses();
    return 0;
}
```

**執行結果：**

```
$ gcc -o memmap memmap.c && ./memmap
=== 記憶體位址分佈 ===
[Text]  函式位址:     0x5638a1234650
[Data]  全域已初始化: 0x5638a1237010  值=42
[Data]  靜態變數:     0x5638a1237014  值=99
[BSS]   全域未初始化: 0x5638a1237018  值=0
[Heap]  malloc 配置:  0x5638a2e40260  值=777
[Stack] 區域變數:     0x7ffe3b821a3c  值=10
```

> ⚠️ **觀察重點**：Stack 位址（`0x7ffe...`）遠高於 Heap（`0x5638...`），且 Stack 往低位址成長，Heap 往高位址成長，兩者從兩端向中間擴展。

## 為什麼 Heap 往上、Stack 往下？

這種設計最大化利用可用空間：

- Stack 和 Heap 之間的未使用空間是「彈性緩衝區」
- 如果其中一個需要更多空間，可以往中間擴張而不需固定限制
- 當兩者相撞時，表示記憶體不足（Stack Overflow 或 Heap 耗盡）

## 思考題

> **Q**：為什麼 Stack 的位址在 64 位元系統上以 `0x7ff` 開頭？  
> **A**：這是由於 Linux 的虛擬記憶體配置策略。核心保留高位址空間，使用者空間則分配在低位址。通常使用者空間的上限是 `0x7fffffffffff`（約 128 TB），Stack 從接近這個上限的位置開始往下成長。
