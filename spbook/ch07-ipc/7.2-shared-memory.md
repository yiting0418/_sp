# 7.2 共享記憶體與信號量

## 共享記憶體（Shared Memory）

共享記憶體是**最快的 IPC 機制**——多個程序直接讀寫同一塊物理記憶體，無需核心介入資料傳輸。

### POSIX 共享記憶體 API

| 函式 | 說明 |
|------|------|
| `shm_open(name, flags, mode)` | 建立或開啟共享記憶體物件 |
| `ftruncate(fd, size)` | 設定大小 |
| `mmap(NULL, size, prot, flags, fd, 0)` | 映射到程序位址空間 |
| `munmap(addr, size)` | 取消映射 |
| `shm_unlink(name)` | 刪除共享記憶體物件 |

### 範例：雙程序共享計數器

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/mman.h>
#include <sys/wait.h>
#include <fcntl.h>
#include <semaphore.h>
#include <string.h>

#define SHM_SIZE 4096

int main() {
    // 建立共享記憶體
    int fd = shm_open("/myshm", O_CREAT | O_RDWR, 0644);
    ftruncate(fd, SHM_SIZE);

    // 映射到程序位址空間
    int *shared = (int *)mmap(NULL, SHM_SIZE,
        PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    *shared = 0;

    // 建立信號量
    sem_t *sem = sem_open("/mysem", O_CREAT, 0644, 1);

    pid_t pid = fork();

    for (int i = 0; i < 100000; i++) {
        sem_wait(sem);    // P 操作：減 1（等待）
        (*shared)++;      // 臨界區間
        sem_post(sem);    // V 操作：加 1（釋放）
    }

    if (pid == 0) {
        // 子程序
        munmap(shared, SHM_SIZE);
        close(fd);
        exit(0);
    }

    wait(NULL);
    printf("共享記憶體結果: %d (期望 200000)\n", *shared);

    // 清理
    munmap(shared, SHM_SIZE);
    close(fd);
    shm_unlink("/myshm");
    sem_close(sem);
    sem_unlink("/mysem");
    return 0;
}
```

**編譯執行：**

```bash
$ gcc -pthread -o shm_demo shm_demo.c -lrt && ./shm_demo
共享記憶體結果: 200000 (期望 200000)
```

## 信號量（Semaphore）

信號量是一個整數計數器，支援兩種原子操作：

| 操作 | 別名 | 效果 |
|------|------|------|
| `sem_wait(sem)` | P 操作 | 如果 sem > 0 則減 1，否則阻塞 |
| `sem_post(sem)` | V 操作 | sem 加 1，喚醒等待的執行緒 |

### 二元信號量 vs Mutex

| 特性 | Mutex | 二元信號量 |
|------|-------|-----------|
| 所有權 | 由鎖定的執行緒解鎖 | 任何執行緒都可 post |
| 使用場景 | 保護臨界區間 | 事件通知、資源計數 |
| 遞迴鎖 | 可設定 | 無 |

### 有名信號量 vs 無名信號量

| 類型 | 建立方式 | 適用 |
|------|---------|------|
| **有名（Named）** | `sem_open()` | 不同程序間同步 |
| **無名（Anonymous）** | `sem_init()` | 同一程序的執行緒間或共享記憶體中 |

## IPC 效能比較

```
速度（越快越好）:
  共享記憶體  >>>  Pipe  ≈  FIFO  >  訊息佇列  >  Socket（本地）>  Socket（網路）
```

共享記憶體之所以最快，是因為資料只在使用者空間操作，不需系統呼叫（除了初始的 `mmap()`），也不需核心複製資料。

## 思考題

> **Q**：共享記憶體比 Pipe 快這麼多，為什麼不總是用它？  
> **A**：共享記憶體需要手動同步（信號量），設計不當容易產生競態條件或死結。而 Pipe 自帶阻塞語意——讀取空 pipe 自然等待。此外，共享記憶體只能在同一台機器上使用（而網路 Socket 可跨機器）。選擇 IPC 機制時，要在效能、安全性和開發複雜度間取得平衡。
