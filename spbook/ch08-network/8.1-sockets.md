# 8.1 Socket 基礎

## 什麼是 Socket？

Socket（通訊端點）是網路通訊的抽象介面。應用程式透過 Socket 傳送與接收資料，不需要知道底層網路協定的細節。

```
┌───── 應用程式 ─────┐
│  Server            │        ┌───── 應用程式 ─────┐
│  socket()          │        │  Client            │
│  bind()            │        │  socket()          │
│  listen()          │        │  connect()         │
│  accept() ←── TCP ────────│  send()/recv()     │
│  send()/recv()     │        │  close()           │
│  close()           │        └────────────────────┘
└────────────────────┘
```

## TCP vs UDP

| 特性 | TCP | UDP |
|------|-----|-----|
| 連線 | 面向連線（先建立連線） | 無連線（直接發送） |
| 可靠性 | 保證送達、順序正確 | 不保證送達 |
| 速度 | 慢（有確認/重送機制） | 快（沒有開銷） |
| 適合場景 | 網頁、Email、檔案傳輸 | 串流、遊戲、DNS |
| 資料邊界 | 串流（Stream） | 封包（Datagram） |

## Socket 位址結構（IPv4）

```c
#include <arpa/inet.h>

// IPv4 Socket 位址結構
struct sockaddr_in {
    sa_family_t    sin_family;  // AF_INET
    in_port_t      sin_port;    // 埠號（network byte order）
    struct in_addr sin_addr;    // IP 位址
    char           sin_zero[8]; // 填充
};

// IP 位址結構
struct in_addr {
    uint32_t s_addr;  // network byte order
};
```

## 位元組序（Byte Order）

網路使用 **Big-Endian（大端序）**，而 x86/ARM 處理器使用 **Little-Endian（小端序）**。轉換函式如下：

```c
uint32_t htonl(uint32_t hostlong);  // host → network (32-bit)
uint16_t htons(uint16_t hostshort); // host → network (16-bit)
uint32_t ntohl(uint32_t netlong);   // network → host (32-bit)
uint16_t ntohs(uint16_t netshort);  // network → host (16-bit)
```

## 位址轉換

```c
#include <arpa/inet.h>

// 點分十進制 → 二進位 (e.g., "192.168.1.1" → 0xC0A80101)
inet_pton(AF_INET, "192.168.1.1", &addr.sin_addr);

// 二進位 → 點分十進制
char str[INET_ADDRSTRLEN];
inet_ntop(AF_INET, &addr.sin_addr, str, sizeof(str));
printf("%s\n", str);  // "192.168.1.1"
```

## 思考題

> **Q**：為什麼網路通訊使用 Big-Endian 而 x86 使用 Little-Endian？  
> **A**：歷史原因——網路協定在 Little-Endian 處理器普及之前就已標準化為 Big-Endian。使用統一的網路位元組序（Network Byte Order），確保不同架構的機器之間可以正確交換資料。所有 POSIX 系統都必須實作 htonl/ntohl 等轉換函式。
