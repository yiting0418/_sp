# 1.3 中斷處理

## 什麼是中斷？

中斷（Interrupt）是一種讓硬體或軟體可以非同步通知 CPU 的機制。當 CPU 收到中斷訊號時，它會：

1. 暫停當前執行的工作
2. 保存當前的 CPU 狀態
3. 跳轉到對應的**中斷處理常式（Interrupt Service Routine, ISR）**
4. 處理完後恢復原本的工作

## 中斷類型

| 類型 | 來源 | 舉例 |
|------|------|------|
| **硬體中斷** | 外部裝置 | 鍵盤輸入、網路封包到達、磁碟 I/O 完成 |
| **軟體中斷** | 程式指令 | `int 0x80`（Linux 系統呼叫傳統方式） |
| **異常（Exception）** | CPU 內部 | 除零錯誤、Page Fault、無效指令 |

## Linux 信號（Signal）機制

Linux 的 Signal 是在使用者空間層級模擬中斷的機制。當行程收到信號時，會暫停當前執行，跳轉到註冊的信號處理器。

### 程式範例：攔截 SIGINT

```c
#include <signal.h>
#include <stdio.h>
#include <unistd.h>

// 這個函式就像「中斷服務常式（ISR）」
void handle_sigint(int sig) {
    printf("\n捕捉到 Ctrl+C！(信號編號: %d)\n", sig);
    printf("中斷處理完畢，程式繼續...\n");
}

int main() {
    // 安裝信號處理器（類似設定中斷向量）
    signal(SIGINT, handle_sigint);

    printf("程式執行中，按 Ctrl+C 觸發中斷...\n");
    for (int i = 1; i <= 5; i++) {
        printf("  計數: %d\n", i);
        sleep(1);
    }
    return 0;
}
```

**編譯執行：**

```bash
$ gcc -o signal_demo signal_demo.c && ./signal_demo
程式執行中，按 Ctrl+C 觸發中斷...
  計數: 1
  計數: 2
^C
捕捉到 Ctrl+C！(信號編號: 2)
中斷處理完畢，程式繼續...
  計數: 3
  計數: 4
  計數: 5
```

### 重要信號

| 信號 | 預設行為 | 觸發條件 |
|------|---------|---------|
| `SIGINT` (2) | 終止程式 | Ctrl+C |
| `SIGKILL` (9) | 強制終止 | `kill -9 PID`（**無法攔截**） |
| `SIGSEGV` (11) | 終止 + Core Dump | 存取非法記憶體位址 |
| `SIGTERM` (15) | 終止程式 | `kill PID` |
| `SIGCHLD` (17) | 忽略 | 子程序結束時通知父程序 |

### 中斷 vs 信號對照

| 硬體中斷 | Linux 信號 |
|---------|-----------|
| 由硬體產生 | 由核心或程式產生 |
| 由 CPU 硬體處理 | 由核心軟體處理 |
| 中斷向量表 | Signal Handler Table |
| 不可忽略（NMI 除外） | 可攔截、忽略或封鎖 |

## 中斷遮罩

CPU 可以暫時關閉中斷（Disable Interrupts）來保護臨界區段。對應到使用者空間，可以使用 `signprocmask()` 來暫時封鎖特定信號。

## 思考題

> **Q**：`SIGKILL` 為什麼不能被攔截？  
> **A**：如果 `SIGKILL` 可以被攔截，惡意或失控的程式就可以永久忽略終止請求，導致系統管理員無法殺死該程式。因此 `SIGKILL` 和 `SIGSTOP` 是唯二無法被攔截的信號，確保了管理員對系統的控制權。
