# 1.1 作業系統的角色

## 什麼是作業系統？

作業系統（Operating System, OS）介於硬體（Hardware）與使用者程式（User Programs）之間，扮演兩個核心角色：

### 1. 資源管理者（Resource Manager）

OS 負責分配與協調電腦系統中所有資源：

| 資源 | OS 管理方式 |
|------|-------------|
| **CPU** | 排程演算法決定下一個執行的程序 |
| **記憶體** | 虛擬記憶體、分頁、分段 |
| **磁碟** | 檔案系統、I/O 排程 |
| **網路** | 網路堆疊、Socket 管理 |
| **周邊裝置** | 裝置驅動程式、中斷處理 |

### 2. 延伸機器（Extended Machine）

OS 為應用程式提供比裸機硬體更高階、更易使用的抽象：

```
你的程式 (Chrome, VS Code, ...)
    ↕ 系統呼叫介面 (System Call Interface)
┌──────────────────────────────────┐
│     作業系統核心 (Kernel)         │
│  ┌────────┐ ┌──────┐ ┌───────┐ │
│  │ 程序   │ │ 記憶 │ │ 檔案  │ │
│  │ 管理   │ │ 體   │ │ 系統  │ │
│  └────────┘ └──────┘ └───────┘ │
└──────────────────────────────────┘
    ↕ 裝置驅動程式 (Device Drivers)
CPU ── RAM ── SSD ── 網卡 ── GPU
```

## 多工與虛擬化

OS 透過 **多工（Multiplexing）** 讓多個程式共用資源：

- **時間多工**（CPU）：快速切換程式，產生同時執行的錯覺
- **空間多工**（記憶體）：每個程式擁有自己獨立的記憶體空間

## 關鍵設計目標

1. **保護（Protection）** ─ 程式之間不互相干擾
2. **抽象（Abstraction）** ─ 隱藏硬體細節，提供簡潔介面
3. **公平（Fairness）** ─ 資源均等分配
4. **效能（Performance）** ─ 最小化 Overhead

## 思考題

> **Q**：為什麼不讓使用者程式直接控制硬體？  
> **A**：如果任何程式都能直接存取硬體，惡意或有 bug 的程式可能損毀其他程式的資料，甚至讓整個系統崩潰。OS 作為中介層提供保護與隔離。
