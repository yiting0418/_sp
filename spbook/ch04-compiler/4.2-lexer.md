# 4.2 詞法分析器（Lexer）

## 什麼是 Lexer？

Lexer（詞法分析器）是編譯器的第一階段，負責將原始碼的字串切割成一個一個的**Token**：

```
輸入："int sum = 10 + count;"

輸出：
  KEYWORD   "int"
  IDENT     "sum"
  ASSIGN    "="
  NUMBER    "10"
  PLUS      "+"
  IDENT     "count"
  SEMI      ";"
  EOF       ""
```

## 實作一個簡易 C Lexer

```c
#include <stdio.h>
#include <ctype.h>
#include <string.h>

typedef enum {
    TOK_KEYWORD, TOK_IDENT, TOK_NUMBER,
    TOK_PLUS, TOK_MINUS, TOK_MUL, TOK_DIV,
    TOK_ASSIGN, TOK_SEMI, TOK_EOF, TOK_UNKNOWN
} TokenType;

const char *type_name[] = {
    "KEYWORD","IDENT","NUMBER","PLUS","MINUS",
    "MUL","DIV","ASSIGN","SEMI","EOF","UNKNOWN"
};
const char *keywords[] = {"int","if","while","return",NULL};

typedef struct { TokenType type; char value[64]; } Token;
const char *src; int pos;

int is_keyword(const char *s) {
    for (int i = 0; keywords[i]; i++)
        if (strcmp(s, keywords[i]) == 0) return 1;
    return 0;
}

Token next_token() {
    Token t = {TOK_EOF, ""};
    while (src[pos]==' '||src[pos]=='\t') pos++;
    if (!src[pos]) return t;

    if (isalpha(src[pos])) {
        int i = 0;
        while (isalnum(src[pos])) t.value[i++] = src[pos++];
        t.value[i] = 0;
        t.type = is_keyword(t.value) ? TOK_KEYWORD : TOK_IDENT;
    } else if (isdigit(src[pos])) {
        int i = 0;
        while (isdigit(src[pos])) t.value[i++] = src[pos++];
        t.value[i] = 0; t.type = TOK_NUMBER;
    } else {
        t.value[0] = src[pos]; t.value[1] = 0;
        switch(src[pos++]) {
            case '+': t.type=TOK_PLUS; break;
            case '-': t.type=TOK_MINUS; break;
            case '*': t.type=TOK_MUL; break;
            case '=': t.type=TOK_ASSIGN; break;
            case ';': t.type=TOK_SEMI; break;
            default:  t.type=TOK_UNKNOWN;
        }
    }
    return t;
}

int main() {
    src = "int sum = 10 + count;";
    pos = 0;
    printf("輸入: %s\n\n", src);
    printf("%-10s %-12s\n", "類型", "值");
    printf("%-10s %-12s\n", "----------", "------------");
    Token t;
    do {
        t = next_token();
        printf("%-12s \"%s\"\n", type_name[t.type], t.value);
    } while (t.type != TOK_EOF);
    return 0;
}
```

**執行結果：**

```
$ gcc -o lexer lexer.c && ./lexer
輸入: int sum = 10 + count;

類型         值
---------- ------------
KEYWORD      "int"
IDENT        "sum"
ASSIGN       "="
NUMBER       "10"
PLUS         "+"
IDENT        "count"
SEMI         ";"
EOF          ""
```

## 如何擴充這個 Lexer？

| 功能 | 實作方式 |
|------|---------|
| 字串字面值 | 處理 `"..."` 和 `'...'` 逃脫字元 |
| 註解 | 跳過 `//` 行註解和 `/* */` 塊註解 |
| 更多關鍵字 | 擴充 `keywords` 陣列 |
| 運算子 | 處理 `++`、`==`、`!=` 等雙字元運算子 |
| 錯誤回報 | 遇到無法辨識的字元時輸出錯誤訊息 |

## 思考題

> **Q**：為什麼要將詞法分析和語法分析分成兩個階段？  
> **A**：分離關注點——Lexer 專注於字元層次的處理（簡單、高效、可用正則表達式驅動），Parser 專注於結構層次的處理（遞迴、上下文相關）。這樣的軟體架構更清晰，且 Lexer 可以獨立最佳化（例如使用狀態機）。
