# 4.3 遞迴下降語法分析器

## 什麼是遞迴下降解析？

遞迴下降（Recursive Descent）是一種**自上而下**的語法分析方法。每個語法規則對應一個函式，函式之間可以互相遞迴呼叫，形成對語法樹的深度優先走訪。

## 語法規則（以算術運算式為例）

```
expr    → term (('+' | '-') term)*
term    → factor (('*' | '/') factor)*
factor  → NUMBER | '(' expr ')' | '-' factor
```

## 完整實作

```c
#include <stdio.h>
#include <ctype.h>

const char *expr; int epos;

int parse_expr();

int parse_factor() {
    while (expr[epos]==' ') epos++;
    if (expr[epos] == '(') {
        epos++;  // 跳過 '('
        int v = parse_expr();
        epos++;  // 跳過 ')'
        return v;
    }
    int n = 0;
    int neg = (expr[epos] == '-') ? (epos++, -1) : 1;
    while (isdigit(expr[epos])) n = n*10 + (expr[epos++]-'0');
    return neg * n;
}

int parse_term() {
    int v = parse_factor();
    while (1) {
        while(expr[epos]==' ') epos++;
        if (expr[epos]=='*') { epos++; v *= parse_factor(); }
        else if(expr[epos]=='/') { epos++; v /= parse_factor(); }
        else break;
    }
    return v;
}

int parse_expr() {
    int v = parse_term();
    while (1) {
        while(expr[epos]==' ') epos++;
        if (expr[epos]=='+') { epos++; v += parse_term(); }
        else if(expr[epos]=='-') { epos++; v -= parse_term(); }
        else break;
    }
    return v;
}

void eval(const char *e) {
    expr = e; epos = 0;
    printf("  %s = %d\n", e, parse_expr());
}

int main() {
    printf("=== 遞迴下降運算式計算器 ===\n");
    eval("3 + 4");
    eval("3 + 4 * 2");       // 優先順序
    eval("(3 + 4) * 2");     // 括號
    eval("100 - 20 * 3 + 5");
    eval("(10 + 5) * (4 - 1)");
    eval("2 * 3 * 4 * 5");
    return 0;
}
```

**執行結果：**

```
$ gcc -o parser parser.c && ./parser
=== 遞迴下降運算式計算器 ===
  3 + 4 = 7
  3 + 4 * 2 = 11
  (3 + 4) * 2 = 14
  100 - 20 * 3 + 5 = 45
  (10 + 5) * (4 - 1) = 45
  2 * 3 * 4 * 5 = 120
```

## 解析過程追蹤

以 `3 + 4 * 2` 為例：

```
parse_expr()
  ├── parse_term()
  │     ├── parse_factor() → 3
  │     └── (no * or /), 回傳 3
  ├── '+' 找到
  ├── parse_term()
  │     ├── parse_factor() → 4
  │     ├── '*' 找到
  │     ├── parse_factor() → 2
  │     └── 回傳 4*2 = 8
  └── 回傳 3+8 = 11
```

## 運算子優先順序的實現

關鍵在於**解析層級**：

```
expr (最低優先權)  ← 處理 + -
  ↓ 呼叫
term (中等優先權)  ← 處理 * /
  ↓ 呼叫
factor (最高優先權) ← 處理數字、括號、負號
```

越底層的函式處理的運算子優先權越高，因此 `*` 會比 `+` 先被計算。

## 思考題

> **Q**：這個計算器如何處理 `3 - 4 - 5`？結果是 `(3-4)-5 = -6` 還是 `3-(4-5) = 4`？  
> **A**：`parse_expr()` 使用左遞迴迴圈處理加減法，遇到 `-` 時立即減去 `parse_term()` 的值，因此 `3 - 4 - 5` 等同於 `(3 - 4) - 5 = -6`，符合左結合性（Left Associativity），這是大部分程式語言的行為。
